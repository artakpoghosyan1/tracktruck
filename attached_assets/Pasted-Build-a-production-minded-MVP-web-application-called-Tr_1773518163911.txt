Build a production-minded MVP web application called TrackTruck Live.
The application allows an admin user to create a truck route on a map, activate the route by paying, start the route simulation, and then share a public link where viewers can watch the truck move live along the route.
The system simulates a truck traveling along a predefined route and ensures that all viewers see the same truck position at the same time, even if they open the link later.
Domain structure
The application will run on:
Admin panel
 tracktruck.live/admin
Public route pages
 tracktruck.live/[id]
Where [id] is a share token generated when a route is activated after payment.
Admin users manage routes from /admin.
 Public viewers open /[id] to watch the truck move live.

Business rules
A route may exist as draft without payment.
Payment is required before a public share link is generated.
Once payment succeeds:
a share token is created


the route becomes ready to start.


When a route finishes:
its share token becomes invalid


the public link stops working.


If the admin wants to run the same route again:
they must pay again


a new share token is generated.


Deleting a route invalidates any share links.

Live simulation system
This application simulates a truck traveling along the route.
All viewers must see the same truck position.
Important rule
Do NOT store every coordinate every second in the database.
Instead use a simulation model.
Persistent data stored in PostgreSQL
Route geometry
 Stops
 Route revisions
 Simulation state
 Payment records
 Share links

Redis live state
Redis should store one object per active route.
Do NOT store "current position" separately.
Instead store a single object called latest_live_snapshot.
Redis key format
route_live_snapshot:{route_id}
Example value
{
 routeId: "123",
 timestamp: "...",
 status: "IN_PROGRESS",
 distanceTraveledM: 30210,
 progressPercent: 41,
 lat: 40.1832,
 lng: 44.5151,
 bearing: 90
}
This snapshot already contains the truck’s current position.
The snapshot should be updated every few seconds.
Redis acts only as a fast cache for the latest state, not as the system of record.
If Redis is empty, the backend must recompute the position from the database state.

Position calculation
Truck position must be computed from
Route polyline
 Truck speed
 Stops and stop durations
 Start time
 Pause time
 Simulation status
Formula concept
distance_traveled = speed × effective_movement_time
Then determine the corresponding point on the route polyline.
